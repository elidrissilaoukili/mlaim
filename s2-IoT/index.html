Methodology & Implementation Steps

Step 1: Initial Approach with Object Detection

We started with object detection because it is simpler for training and testing. Object detection helped us understand the system integration but wasn't sufficient for secure access control.

Step 2: Background Removal Experiment

We attempted to remove backgrounds from images thinking it would improve accuracy, but this approach did not yield the expected improvements.

Step 3: Data Collection Strategy

Started collecting images of authorized faces and random faces for unauthorized classification. This transition to face classification provided better security control.

Step 4: Model Training on Edge Impulse

Uploaded the collected data to Edge Impulse platform and trained the face recognition model, followed by initial testing phases.

Step 5: Camera Quality Discovery

Key Learning: We discovered that using different cameras for training (phone) and testing (webcam) resulted in poor model performance. This highlighted the importance of collecting data from various camera types and qualities.

Step 6: Model Deployment

After achieving satisfactory model performance, we deployed the trained model to the ESP32-CAM device.

Step 7: Serial Monitor Testing

Conducted simple testing for face recognition, printing results on the serial monitor. The system successfully detected authorized and unauthorized faces.

Step 8: LED Integration

Updated the Arduino code to control LED indicators: blue lamp for authorized faces and yellow lamp for unauthorized faces.

Step 9: Servo Motor Integration

Added servo motor functionality to simulate door opening mechanism - opening when face is authorized and remaining closed for unauthorized access.

Step 10: System Documentation

Recorded demonstration video showing the complete system functionality and documented the entire development process.
